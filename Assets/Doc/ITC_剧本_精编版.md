#### 【新手教学环节】

● **[13号窗口老员工]**

接着，客户会递交申请。点击气动通道接收文件，拖动到桌面查看。仔细检查：
1. **封蜡是否破损** —— 若破坏，表明有人擅自查看，必须拒绝

5. **身份核实** —— 对比肖像与本人，注意变装迹象

- 把文件拖入打字机

- 三次错误：签约失败

### 核心分支逻辑

#### 签约流程分支

```
START
  ├─ DAY == 1 → 客户序列1
  ├─ DAY == 2 → 客户序列2
  └─ DAY >= 3 → 后续客户序列

每位客户流程：
  1. 接收文件
  2. 【分支】是否退回文件？
     ├─ 退回 → 选择理由 → 判定是否正确
     └─ 不退回 → 继续
  3. 打字机QTE
     └─ 失误累计
  4. 【分支】选择印章
     ├─ 选错 → Sign_mistake = 1
     └─ 选对 → 继续
  5. 盖章时机
     ├─ 最佳 → satisfaction + 1
     └─ 过早/过晚 → 无变化
  6. 灵魂收取QTE
     ├─ 收取过多 → satisfaction - 1
     ├─ 收取正确 → 无变化
     └─ 收取过少 → Sign_mistake = 1
  7. Henet电话反馈
     └─ 根据 Sign_mistake 和 satisfaction 触发不同对话
```

#### 需要外部路由的交互判定节点

以下节点需要游戏系统在进入前设置路由变量：

| 节点ID | 描述 | 需要设置的变量 |
|--------|------|----------------|
| C2_D146 | 选取印章正确/错误 | 玩家选择的印章类型 |
| C2_D155 | 判定区域错误次数 | QTE判定结果 |
| C3_D176 | 收取部分比例阈值 | 灵魂收取百分比 |
| C3_D252 | 满意度阈值判定 | 当前satisfaction值 |
