# Daoyangao Generator（导演稿生成）

将“薄台词”扩写为可执行的演出指导稿：不改动原句，只做节拍拆分 + 动作动词（Actioning）+ 潜台词 + 五维韵律标注（停顿、语速、情绪档位、句尾收束、强调词）。

## 输入（先要这些；缺失则先问）

优先收集最少信息（缺哪个问哪个；用户不给就写明你的默认假设）：

- 说话者/对话对象（或“对空气/自言自语”）
- 关系与地位：谁更强势？谁更怕失去？
- 场景一句话：当下压力/触发点是什么
- 即时目标：这句话想改变对方什么（让他答应/退让/羞愧/暴露）
- 风险：说错一句会怎样（被杀/关系崩/任务失败）
- 原句台词（必须逐字保留）

推荐用户按这个格式贴给你：

```text
角色：
对话对象：
关系/地位：
场景：
即时目标：
风险：
台词：
- （逐字原句1）
- （逐字原句2）
```

## 默认输出（人类可读的导演稿，逐句产出）

对每句台词强制输出四块：

1) 信息栏（1句话，含：角色/对象/关系地位/场景压力/即时目标）
2) 节拍表（B1…Bn；每拍必写：动作动词 + 潜台词(可选) + 五维）
3) 正文标注版台词（原句逐字保留，只插入极简符号/括注）
4) 备选演法（最多2条；如“更克制版/更讽刺版”；仍不改原句）

输出骨架（可直接复制）：

也可直接拷贝并填写 `assets/director-sheet-template.md`。

```md
### （角色 → 对话对象）台词：<原句>
**信息栏**：<一句话>

**节拍表**
- B1 〔动词：To ___〕〔G?〕〔速?〕（潜台词：...）强调：【...】收束：↓/↑/↘/↘↗ 停顿：?微/?短/?中/?长/?静
- B2 ...

**正文（标注版；原句逐字保留）**
<带 ?停顿? / 〔速/档位〕 / 【强调】 / 行末收束 的原句>

**备选演法（≤2）**
- 备选A：<一句话说明差异>；正文：<仍是不改原句的标注版>
```

## 极简符号（不要扩成DSL）

只允许在“正文标注版台词”里插入以下符号：

### 停顿（? ?）

- `?微?` = 0.10s（短语边界微停）
- `?短?` = 0.30s（逗号级）
- `?中?` = 0.70s（句内“想一下/转折前”）
- `?长?` = 1.20s（句末情绪发酵）
- `?静?` = 2.50s（Silence：墙壁）
- 需要精确时：`?0.45?`（单位秒）

强制区分三类“沉默”：

- `...`：还在连接里（找词/犹豫/吞咽）
- Pause：桥梁（常伴随心理活动/节拍转换；可用 `?长?` 表示）
- Silence：墙壁（沟通断裂/认知冲击；用 `?静?`）

### 语速（〔 〕）

- `〔速↑〕` ≈ 1.15 倍（更急、更推进）
- `〔速↑↑〕` ≈ 1.30 倍（机关枪/追问）
- `〔速↓〕` ≈ 0.85 倍（压住、沉着、咬字）
- `〔速↓↓〕` ≈ 0.70 倍（沉重/威压/讽刺拖长）
- 可覆盖：`〔速=0.92〕`

默认约定（不必每次写）：功能词压缩、内容词延展；句末 2~3 个字自然减速一点点。

### 情绪档位（Gear：〔G1~G5〕）

用“能量层级”而不是情绪形容词当指令：

- `〔G1〕` 压抑/虚弱/濒临崩溃
- `〔G2〕` 中性叙述
- `〔G3〕` 投入/有锋芒的正常强度
- `〔G4〕` 高唤醒（争吵/惊吓/兴奋）
- `〔G5〕` 失控边缘（尖叫/彻底爆炸）

### 句尾收束（行末箭头）

- `↓` 落地（陈述/命令/结论/权威）
- `↑` 悬空（真疑问/求确认/示弱）
- `↘` 断切（戛然而止/被打断/强硬终止）
- `↘↗` 回旋（话没说尽/试探/委婉威胁）

规则：修辞疑问默认 `↓`；只有真求确认才用 `↑`。

### 强调词（【】）

- 用 `【重点词】` 标信息焦点/对比/否定/数字/专名/转折落点
- 每拍最多 3 个【】；宁少勿滥（越滥越像播报腔）

## 工作流（按这个顺序做；每拍参数允许不同）

### Step 0：定基线

- 抽取：关系与地位 / 目标 / 风险
- 将用户给的“情绪标签”降级为线索，不当作指令；优先把意图动作化

### Step 1：切节拍（Beat）

按“策略变化/新信息/转折词/省略号/破折号/强标点”切；同一句里只要策略变了就必须新拍。

经验入口：出现“但是/可/所以/别/你听我说”高概率切拍；“先软后硬/先解释后指控/先试探后摊牌”必切。

### Step 2：每拍写动作动词（Actioning）

用及物动词写清方向：我用 X 作用于你。

小型动词库（够用就好）：

- 亲近：To Soothe / To Invite / To Confess / To Charm
- 施压：To Threaten / To Corner / To Command / To Shame
- 防御：To Deflect / To Deny / To Downplay / To Stall
- 探测：To Test / To Probe / To Fish / To Verify
- 断联：To Dismiss / To Cut Off / To End

### Step 3：写潜台词（只写一句）

当字面与真实意图不一致时，写 `（潜台词：…）`，并以潜台词驱动投放（可覆盖字面语义）。

### Step 4：五维落笔（每拍必填）

- 停顿：先按标点与语法断句，再用逻辑/心理停顿补齐；必要时区分 `...`/Pause/`?静?`
- 语速：先定整拍 `〔速↑/↓〕`，再默认 Rubato（功能词快、内容词慢、句末减速）
- 档位：用 `〔G1~G5〕` 量化能量，不写“生气地/害怕地”当指令
- 收束：定行末 `↓/↑/↘/↘↗`（尤其修辞疑问别乱用 `↑`）
- 强调：圈出 1~3 个【重点词】（新信息/对比/否定/数字/专名/转折落点）

### Step 5：写正文标注版台词

- 严格保持原句逐字不变
- 只插：`?停顿?`、`〔速/档位〕`、`【重点词】`、（括注）与行末收束箭头

## 可选输出A：给“文字导演/打字机效果”的节奏轨

仅在用户明确需要时输出（放在每句台词后，作为附录即可）：

- 将同一套“节拍 + 五维”映射为 typewriter/kinetic 参数
- 将停顿 `?微/?短/?中/?长/?静` 映射为逐字显示的延迟/整句停住/清空文本框等行为
- 将 `〔速↑/↓〕` 映射为每秒显示字符数（CPS）倍率；将 `【强调】` 映射为短词的轻量强调动效
- 遵守无障碍：允许调速/跳过；提供 Reduce Motion 兜底；避免让长段落整体抖动

## 可选输出B：引擎可读JSON（严格模式）

仅在用户明确需要“机器可读/可落地”时输出：

- 输出纯 JSON（不加 Markdown、不加解释、不加注释）
- 使用白名单字段与白名单标签（避免生成引擎不支持的效果）

建议字段：

- `line_id`、`speaker`、`addressee`
- `original_text`
- `beats[]`: `id`、`action`、`subtext`、`prosody`（pause/rate/gear/closure/emphasis）、`annotated_text`
- 可选 `typewriter`（cps/holds/effects）

## 质量自检（交付前必做）

- 原句逐字保留：没有改字、没有同义替换
- 节拍合理：策略变了就切拍；每拍参数允许不同
- 动作化优先：每拍有清晰及物动词；少用情绪形容词
- 潜台词到位：反讽/礼貌攻击/压抑式愤怒等必须写潜台词并以其驱动投放
- 五维齐全：每拍都填停顿/语速/档位/收束/强调；强调词不超过 3 个
- 箭头不乱：修辞疑问默认 `↓`；被打断/戛然而止用 `↘`
